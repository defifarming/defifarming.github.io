Network Structure

To function properly, the Dexit protocol requires several components. 

Full Node

Full nodes are overloaded with data, because of this they require more advanced computing power and are energy-intensive, which means they are expensive. They are essential for general safety.

Light Node

Light node means you are ready to send queries to other nodes. They might be slow and small amount which support light node peers. If your tasks don’t rely from critical fast query processing then you can look for light node.

Archive Node

The archive node on the hard disk can takes lot of spaces, The node stores not only the most recent state but also the entire history of network events between blocks.

Validators

Validators are responsible for validating transactions and generating blocks, ensuring the network’s security and the consistency of the ledger. In return, they receive rewards from the gas Fee of transactions.

Staking

It is responsible for holding the staking function to determine validators through independent election, and the election workflow are performed via staking procedure.

Voting

Voting is a mechanism where validator can create a proposal to change the size of maximum validator at any given time and minimum staking amount in which all active validator will vote to approve the proposal.

The Dexit Token

The DXT token is a unit of value that is native to the Dexit network. It represents a medium of exchange between users & developers who pay to use the network and validators who take the fees as payment for the services they provide.
